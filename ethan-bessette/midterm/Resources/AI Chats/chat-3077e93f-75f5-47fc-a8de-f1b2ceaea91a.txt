### PyTorch `play`

This version accepts either a `torch.Tensor` or something NumPy-convertible.

```python
def play_torch(audio, sample_rate):
    """Play audio in a notebook using HTML5 audio widget."""
    if isinstance(audio, torch.Tensor):
        audio = audio.detach().cpu()

        if audio.ndim == 2:
            audio = audio[0]

        audio = audio.clamp(-1.0, 1.0)
        audio_np = audio.numpy()
    else:
        audio_np = np.asarray(audio)
        if audio_np.ndim == 2:
            audio_np = audio_np[0]

    normalizer = float(np.iinfo(np.int16).max)
    audio_int16 = (audio_np * normalizer).astype(np.int16)

    memfile = io.BytesIO()
    wavfile.write(memfile, sample_rate, audio_int16)

    html = """<audio controls>
                <source src="data:audio/wav;base64,{base64_wavfile}" type="audio/wav" />
                Your browser does not support the audio element.
              </audio>"""

    html = html.format(
        base64_wavfile=base64.b64encode(memfile.getvalue()).decode("ascii")
    )
    memfile.close()
    display.display(display.HTML(html))
```

### PyTorch `specplot`

PyTorch has `torch.stft`, so this version avoids `scipy.signal.stft`.

```python
def specplot_torch(audio, sample_rate, vmin=-5, vmax=1, rotate=True, size=512 + 256):
    """Plot the log magnitude spectrogram of audio using torch.stft."""
    if not isinstance(audio, torch.Tensor):
        audio = torch.tensor(audio, dtype=torch.float32)
    else:
        audio = audio.detach().cpu().float()

    if audio.ndim == 2:
        audio = audio[0]

    hop_length = size // 4
    window = torch.hann_window(size)

    stft = torch.stft(
        audio,
        n_fft=size,
        hop_length=hop_length,
        win_length=size,
        window=window,
        center=True,
        return_complex=True,
    )

    logmag = torch.log10(torch.abs(stft) + 1e-7)

    if rotate:
        logmag = torch.flip(logmag, dims=[0])

    plt.matshow(logmag.numpy(), vmin=vmin, vmax=vmax, cmap=plt.cm.magma, aspect="auto")
    plt.xticks([])
    plt.yticks([])
    plt.xlabel("Time")
    plt.ylabel("Frequency")
```

---

### Notes

- `play_torch()` uses PyTorch only for tensor handling, but `wavfile.write`, `io`, and `base64` are still the right tools for outputting playable audio in a notebook.
- `specplot_torch()` is the more direct PyTorch conversion because `torch.stft()` replaces `scipy.signal.stft()`.
- If your tensors are on GPU, these functions move them to CPU with `.detach().cpu()` before plotting or writing audio.

```python
import time
import torch
import numpy as np
import sounddevice as sd


def play_audio(audio, sample_rate):
    if isinstance(audio, torch.Tensor):
        audio = audio.detach().cpu().float().numpy()
    else:
        audio = np.asarray(audio, dtype=np.float32)

    if audio.ndim == 2:
        audio = audio[0]

    audio = np.clip(audio, -1.0, 1.0)
```

```python
import torch
import matplotlib.pyplot as plt


def specplot_torch(audio, sample_rate, vmin=-5, vmax=1, rotate=True, size=768):
    """Plot the log-magnitude spectrogram of audio using torch.stft and matplotlib.pyplot."""
    if not isinstance(audio, torch.Tensor):
        audio = torch.tensor(audio, dtype=torch.float32)
    else:
        audio = audio.detach().cpu().float()

    if audio.ndim == 2:
        audio = audio[0]

    hop_length = size // 4
    window = torch.hann_window(size)

    stft = torch.stft(
        audio,
        n_fft=size,
        hop_length=hop_length,
        win_length=size,
        window=window,
        center=True,
        return_complex=True,
    )

    logmag = torch.log10(torch.abs(stft) + 1e-7)

    if rotate:
        logmag = torch.flip(logmag, dims=[0])

    logmag = logmag.numpy()

    plt.figure(figsize=(10, 4))
    plt.imshow(
        logmag,
        vmin=vmin,
        vmax=vmax,
        cmap="magma",
        aspect="auto",
        origin="upper" if rotate else "lower",
    )
    plt.xlabel("Time")
    plt.ylabel("Frequency")
    plt.xticks([])
    plt.yticks([])
    plt.tight_layout()
    plt.show()
```

The best way is:

- keep **one fixed ordered list** of parameter names
- store only the **values** in a list / tensor
- when decoding, recover names by **zipping** the values back with that same ordered list

You do **not** need to store names with every example, as long as the order never changes.

## Best structure

```python
PARAMS = [
    "i1ratio", "i1index1", "i1index2", "i2index2", "i2index1",
    "i1out", "i2out",
    "iattack1", "idecay1", "isustaintime1", "isustain1", "irelease1",
    "iattack2", "idecay2", "isustaintime2", "isustain2", "irelease2",
]

stoi = {name: i for i, name in enumerate(PARAMS)}
itos = {i: name for i, name in enumerate(PARAMS)}
```

Note: `stoi` should usually map to **int**, not `float`.

---

## Encode: generate values in fixed order

If you want one random value per parameter:

```python
values = [rnd(1) for _ in PARAMS]
```

Now:

- `values[0]` always means `"i1ratio"`
- `values[1]` always means `"i1index1"`
- etc.

That’s the simplest and best approach.

---

## Decode: assign names back to values

Later, after processing:

```python
decoded = dict(zip(PARAMS, values))
```

Now `decoded` becomes:

```python
{
    "i1ratio": ...,
    "i1index1": ...,
    ...
}
```

That is usually the cleanest way to restore parameter names.

---

## If you want to access by name and by index

You can do both:

```python
i = stoi["i1ratio"]
value = values[i]
```

and reverse:

```python
name = itos[0]
```

---

## Good pattern for ML

For machine learning, this is especially nice because your model outputs a vector:

```python
pred = model(x)   # shape [17]
```

and then you decode with:

```python
pred_params = dict(zip(PARAMS, pred.tolist()))
```

So the model only deals with numeric vectors, and you only reattach names when needed.

---

## Recommended helper functions

```python
PARAMS = [
    "i1ratio", "i1index1", "i1index2", "i2index2", "i2index1",
    "i1out", "i2out",
    "iattack1", "idecay1", "isustaintime1", "isustain1", "irelease1",
    "iattack2", "idecay2", "isustaintime2", "isustain2", "irelease2",
]

stoi = {name: i for i, name in enumerate(PARAMS)}
itos = {i: name for i, name in enumerate(PARAMS)}


def encode_random():
    return [rnd(1) for _ in PARAMS]


def decode_values(values):
    return dict(zip(PARAMS, values))
```

Usage:

```python
encoded = encode_random()
decoded = decode_values(encoded)
```

---

## If using PyTorch tensors

If you want training data as tensors:

```python
import torch

values = torch.tensor([rnd(1) for _ in PARAMS], dtype=torch.float32)
decoded = dict(zip(PARAMS, values.tolist()))
```

That is probably the best setup for your project.

---

## Important rule

This only works well if:

- `PARAMS` is your **single source of truth**
- you never reorder it accidentally

If you change the order later, old encoded vectors will decode incorrectly.

```python
from pathlib import Path


def encode_from_file(file_number: int):
    path = Path(f"Generated/{file_number:04d}.txt")

    with path.open("r") as f:
        lines = f.readlines()

    header_end = "---------END OF HEADER--------------\n"

    try:
        start = lines.index(header_end) + 1
    except ValueError:
        raise ValueError(f"Header end marker not found in {path}")

    values = []
    for line in lines[start:]:
        line = line.strip()
        if not line:
            continue
        values.append(float(line))
        if len(values) == 17:
            break

    if len(values) != 17:
        raise ValueError(f"Expected 17 values in {path}, found {len(values)}")

    return values
```

## More robust version

If the header line might have slightly different spacing/newline formatting, use this instead:

```python
from pathlib import Path


def encode_from_file(file_number: int):
    path = Path(f"Generated/{file_number:04d}.txt")

    with path.open("r") as f:
        lines = f.readlines()

    start = None
    for i, line in enumerate(lines):
        if line.strip() == "---------END OF HEADER--------------":
            start = i + 1
            break

    if start is None:
        raise ValueError(f"Header end marker not found in {path}")

    values = []
    for line in lines[start:]:
        line = line.strip()
        if not line:
            continue
        values.append(float(line))
        if len(values) == 17:
            break

    if len(values) != 17:
        raise ValueError(f"Expected 17 values in {path}, found {len(values)}")

    return values
```

```python
#%%
import torch
import math

#%%
def adsr(a: float, d: float, s_time: float, sustain_level: float, r: float, num_samples: int, sample_rate: int):
    adstr = torch.tensor([a,d,s_time,r])
    adstr = torch.softmax(adstr, dim=0)
    attack = torch.linspace(0.0,1.0, int(num_samples*adstr[0]))
    decay = torch.linspace(1.0, sustain_level, int(num_samples*adstr[1]))
    sustain = torch.linspace(sustain_level, sustain_level, int(num_samples*adstr[2]))

    parameters = [attack, decay, sustain]
    envelope = torch.concat(parameters, dim=0)
    release = torch.linspace(sustain_level, 0.0, num_samples-len(envelope))
    envelope = torch.concat([envelope, release], dim=0)

    return envelope

#%%
def linear_lookup(phase: float, wavetable: torch.Tensor):
    phase = float(phase) % 1.0

    extended = torch.cat([wavetable, wavetable[:1]], dim=0)
    table_size = wavetable.shape[0]

    index = phase * table_size
    i0 = int(math.floor(index))
    i1 = i0 + 1
    frac = index - i0

    return extended[i0] * (1.0 - frac) + extended[i1] * frac


#%%
def fm_bank(base_freq: float, ratio1: float, ratio2: float, i1index2: float, i2index1: float, iout1: float, iout2: float, adsr1: torch.Tensor, adsr2: torch.Tensor, wavetable: torch.Tensor, num_samples: int, sample_rate: int):
    ratio1 = ratio1 * 16
    i1index2 = i1index2 * 14.5
    i2index1 = i2index1 * 14.5

    a1 = torch.zeros((num_samples))
    a2 = torch.zeros((num_samples))
    phase1 = 0
    phase2 = 0

    for sample in range(num_samples):
        if sample == 0:
            freq1 = base_freq * ratio1
            freq2 = base_freq * ratio2
        else:
            freq1 = (base_freq * ratio1)*(1 + float(a2[sample-1]) * i2index1)
            freq2 = (base_freq * ratio2)*(1 + float(a1[sample-1]) * i1index2)

        if sample % 1000 == 0:
            print(freq1, freq2)

        a1[sample] = linear_lookup(phase1, wavetable)
        a1[sample] = a1[sample] * adsr1[sample]

        a2[sample] = linear_lookup(phase2, wavetable)
        a2[sample] = a2[sample] * adsr2[sample]

        phase1 = (phase1 + freq1 / float(sample_rate)) % 1.0
        phase2 = (phase2 + freq2 / float(sample_rate)) % 1.0

    aout = a1 * (iout1 / (iout1 + iout2)) + a2 * (iout2 / (iout1 + iout2))
    return aout
```

```python
#%%
import torch
import math

#%%
def adsr(a: float, d: float, s_time: float, sustain_level: float, r: float, num_samples: int, sample_rate: int):
    adstr = torch.tensor([a,d,s_time,r])
    adstr = torch.softmax(adstr, dim=0)
    attack = torch.linspace(0.0,1.0, int(num_samples*adstr[0]))
    decay = torch.linspace(1.0, sustain_level, int(num_samples*adstr[1]))
    sustain = torch.linspace(sustain_level, sustain_level, int(num_samples*adstr[2]))

    parameters = [attack, decay, sustain]
    envelope = torch.concat(parameters, dim=0)
    release = torch.linspace(sustain_level, 0.0, num_samples-len(envelope))
    envelope = torch.concat([envelope, release], dim=0)

    return envelope

#%%
def linear_lookup(phase: float, wavetable: torch.Tensor):
    phase = float(phase) % 1.0

    extended = torch.cat([wavetable, wavetable[:1]], dim=0)
    table_size = wavetable.shape[0]

    index = phase * table_size
    i0 = int(math.floor(index))
    i1 = i0 + 1
    frac = index - i0

    return extended[i0] * (1.0 - frac) + extended[i1] * frac


#%%
def fm_bank(base_freq: float, ratio1: float, ratio2: float, i1index2: float, i2index1: float, iout1: float, iout2: float, adsr1: torch.Tensor, adsr2: torch.Tensor, wavetable: torch.Tensor, num_samples: int, sample_rate: int):
    ratio1 = ratio1 * 16
    i1index2 = i1index2 * 14.5
    i2index1 = i2index1 * 14.5

    a1 = torch.zeros((num_samples))
    a2 = torch.zeros((num_samples))
    phase1 = 0
    phase2 = 0

    for sample in range(num_samples):
        if sample == 0:
            freq1 = base_freq * ratio1
            freq2 = base_freq * ratio2
        else:
            freq1 = (base_freq * ratio1)*(1 + float(a2[sample-1]) * i2index1)
            freq2 = (base_freq * ratio2)*(1 + float(a1[sample-1]) * i1index2)

        if sample % 1000 == 0:
            print(freq1, freq2)

        a1[sample] = linear_lookup(phase1, wavetable)
        a1[sample] = a1[sample] * adsr1[sample]

        a2[sample] = linear_lookup(phase2, wavetable)
        a2[sample] = a2[sample] * adsr2[sample]

        phase1 = (phase1 + freq1 / float(sample_rate)) % 1.0
        phase2 = (phase2 + freq2 / float(sample_rate)) % 1.0

    aout = a1 * (iout1 / (iout1 + iout2)) + a2 * (iout2 / (iout1 + iout2))
    return aout
```

## Best practical format

Use a **dataset index table + feature files**.

### 1. One metadata table
Store one row per audio example with:

- `audio_path`
- `param_path` or `param_id`
- `midi_note` or `f0`
- `split` (`train` / `test`)
- optionally `num_samples`

This table can be:

- `.csv`
- or better, `.parquet`

Example columns:

```text
example_id,audio_path,param_id,midi_note,split
0,Generated/0007_57.wav,7,57,train
1,Generated/0007_60.wav,7,60,train
2,Generated/0007_64.wav,7,64,test
```

---

### 2. Store targets as a fixed 17-value vector
For ML, your label should be a dense vector:

```python
[ p0, p1, p2, ..., p16 ]
```

not parameter names.

Since many audio files share the same synth parameters, a good structure is:

- one table/dict mapping `param_id -> 17-float vector`
- many audio examples pointing to that `param_id`

Example:

```python
params_by_id[7] = [0.12, 0.88, ..., 0.31]
```

This avoids duplicating labels over and over.

---

### 3. Precompute Mel spectrograms and save them as tensors
Since your model input is the Mel spectrogram, the best training format is usually to preprocess audio once and save:

- `.pt` files if using PyTorch
- or `.npy` / `.npz`

Each feature file contains a tensor shaped like:

```python
[n_mels, time_frames]
```

For your case:

- `n_mels = 128`
- `n_fft = 1024`
- `hop_length = 256`

So each training example becomes:

```python
x = mel_spectrogram    # shape [128, T]
y = parameter_vector   # shape [17]
```

This is much faster than recomputing mels every epoch.

This is the easiest format for your model.

# Why this is better than CSV

A CSV is bad for spectrogram data because:

- spectrograms are 2D tensors
- time dimension may vary
- loading is slow
- parsing huge numeric text files is inefficient
- PyTorch works better with binary tensors

Use binary tensor files instead:
- `.pt`
- `.npy`
- `.npz`

---

# Important design choice: include `f0` in the input or not?

Because multiple audio files share the same parameters and only differ in fundamental frequency, you should strongly consider storing `f0`/`midi_note` with each example.

Why:
- otherwise the model must infer pitch only from the spectrogram
- but the same synth parameters can sound different at different pitches
- giving pitch explicitly may make parameter estimation easier

So each training example should probably be:

```python
input = {
    "mel": [128, T],
    "f0": scalar
}
target = [17]
```

Even if you don’t feed `f0` to the network immediately, keep it in the dataset.

---

# Best split strategy

Since multiple notes share the same parameter set, the split matters.

## Better split
Split by **parameter set**, not by individual audio files.

So:
- all audio derived from one synth parameter vector should go entirely into train or entirely into test

This prevents leakage.

Bad:
- note 57 from param set 7 in train
- note 60 from same param set 7 in test

That would let the model partially memorize the target parameters.

Good:
- param set 7 entirely in train
- param set 8 entirely in test

def adsr(adstsr_numSamples: torch.Tensor):
    a,d,s_time,sustain,r,num_samples = adstsr_numSamples
    adstr = torch.tensor([a,d,s_time,r])
    adstr = torch.softmax(adstr, dim=0)
    attack = torch.linspace(0.0,1.0, int(num_samples*adstr[0]))
    decay = torch.linspace(1.0, sustain, int(num_samples*adstr[1]))
    sustain = torch.linspace(sustain, sustain, int(num_samples*adstr[2]))
    envelope = torch.concat([attack, decay, sustain], dim=0)
    release = torch.linspace(sustain, 0.0, num_samples-len(envelope))
    envelope = torch.concat([envelope, release], dim=0)
    return envelope

==== SimpleCompletableMessage #22 ====
Assistant: 
`torch.cat()` can only concatenate tensors that have the same number of dimensions.

Your problem is likely:

- `parameters[7:11]` is a **1D tensor** of shape `[4]`
- `parameters[17]` is a **0D scalar tensor** of shape `[]`

So this fails:

```python
torch.cat([parameters[7:11], parameters[17]], dim=0)
```

because `[4]` and `[]` do not match.

## Fix

Wrap the scalar in a 1D tensor with one element:

```python
torch.cat([parameters[7:11], parameters[17].unsqueeze(0)], dim=0)
```

That gives:

- `parameters[7:11]` -> shape `[4]`
- `parameters[17].unsqueeze(0)` -> shape `[1]`

so result shape becomes `[5]`.

---

## But your `adsr()` expects 6 values

You said `adsr` expects:

```python
a, d, s_time, sustain, r, num_samples
```

That means you need **6 total values**, not 5.

So you probably want something like:

```python
torch.cat([parameters[7:12], parameters[17].unsqueeze(0)], dim=0)
```

because:

- `parameters[7:12]` gives 5 values:
  - 7
  - 8
  - 9
  - 10
  - 11
- plus `parameters[17]` gives the 6th value

Total = 6 values.

```python
adsr(torch.cat([parameters[7:12], parameters[17].unsqueeze(0)], dim=0))
```

---

## Important indexing note

Python slices exclude the end index:

- `parameters[7:11]` means indices `7, 8, 9, 10`
- `parameters[7:12]` means indices `7, 8, 9, 10, 11`

So if you need attack, decay, sustain time, sustain level, release, you want `7:12`, not `7:11`.

---

## Cleaner version

You could also write:

```python
adsr_params = torch.cat((parameters[7:12], parameters[17].view(1)))
env = adsr(adsr_params)
```

or:

```python
adsr_params = torch.tensor([
    parameters[7],
    parameters[8],

```

But `cat(...view(1))` is cleaner if `parameters` is already a tensor.
