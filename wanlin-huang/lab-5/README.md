# Lab 5: Symbolic Music Generation

## Algorithm Description

This composition uses four algorithmic chapters, each mapping a different computational process to MIDI notes. All chapters share a single random seed (`SEED=42`), meaning the maze shape directly influences the pathfinding route, which in turn determines the sorting array — the entire piece is one interconnected system.

### Chapter 1: DFS Pathfinding
A depth-first search navigates from the top-left corner of a maze to the bottom-right exit. Each step generates one note. Forward steps are louder; backtracking steps are softer with a slightly lower pitch, simulating the tension of exploration and getting lost.

### Chapter 2: Prim's Maze Generation
Prim's algorithm randomly breaks walls to build the maze. Each cell added to the maze generates one note. The column coordinate maps to pitch, and the row depth maps to velocity, producing a sparse and unpredictable construction feel.

### Chapter 3A: Bubble Sort
The column coordinates from the DFS path are extracted as an array and sorted with bubble sort. Each comparison and swap generates a note — comparisons are soft, swaps are loud. The music moves from chaotic to ordered.

### Chapter 3B: Sleep Sort (coda)
The same array is scheduled using Sleep Sort: each element waits a duration proportional to its value before firing. This produces a sparse, ascending coda. A final C major triad resolves the piece.

### Pitch Mapping
All pitches are constrained to the C major scale (`[0, 2, 4, 5, 7, 9, 11]`) spanning two octaves from Middle C, ensuring musical coherence across all chapters.

## Versions and Edits

Four versions were generated and compared during the process:

**V1 — Original**
C major, chapter order: maze → pathfinding → bubble sort → Sleep Sort. Direct algorithm output used as a baseline reference.

**V2 — Closing cadence added**
Built on V1. Modified the Sleep Sort coda by tightening note spacing (`scale_factor` reduced from `0.3` to `0.15`) and adding a final C major triad (`C-E-G`) to resolve the abrupt ending.

**V3 — Pentatonic scale variant**
Changed the scale to pentatonic (`[0, 2, 4, 7, 9]`), `SEED=2025`, `BPM=100`. Produced a distinctly different Eastern feel, demonstrating the algorithm's sensitivity to scale parameters.

**V4 — Final version (submitted)**
Built on V2. Swapped chapter order to: pathfinding → maze → bubble sort → Sleep Sort. This creates a more complete narrative arc — exploring in darkness, a world gradually taking shape, then resolving into order. All notes remain as generated by the algorithm; no manual note editing was performed. Differences between versions arise solely from parameter and ordering choices.
